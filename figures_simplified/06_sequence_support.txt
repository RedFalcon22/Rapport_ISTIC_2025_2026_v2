sequence diagram - an internal user selects a support category and types a question in the Streamlit interface. The message is sent to the FastAPI backend, which first passes it through a guardrail to check if it is allowed. If it passes, the backend corrects spelling mistakes and rewrites the query based on conversation history. It then searches ChromaDB for the 5 most relevant chunks filtered by the selected category. At the same time it runs the emotional detection module to check if the user is in distress. If no distress is detected, the backend checks the Redis cache and if there is no cached answer it calls the LLM server to generate a response. The response is cached in Redis, the session is updated, and the frontend displays the answer along with suggested follow-up questions.
