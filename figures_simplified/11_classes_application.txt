class diagram - the application has 8 classes. A ChatSession holds a session ID, user ID, module type, creation date, and expiry date. It contains multiple Messages, each with an ID, content, role, and timestamp. Each Message is classified by an IntentClassification that holds the intent type and confidence score. A Message can generate a Response that holds the answer content, the source chunks it used, the latency, and whether it was served from cache. A Response references one or more DocumentChunks, each with an ID, text content, embedding vector, category, and source document. A ChatSession can also create a ReservationRequest with an email, phone number, requested date, channel, and subject. A Message can contain an EmotionalSignal with a signal type, severity, and detection time, which is detected by Module 3 and triggers the empathetic flow in Module 2. The HarassmentClassification class is independent from the conversational pipeline and is used only by Module 4; it holds the text ID, a list of labels, the strategy used, confidence score, urgency flag, and timestamp.
