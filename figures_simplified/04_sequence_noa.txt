sequence diagram - a visitor types a question on the Noa chatbot. The React frontend sends the message to the FastAPI backend. The backend classifies the intent as factual, then checks the Redis cache. If the answer is already cached it returns it immediately in less than 1ms. If not, it queries ChromaDB to retrieve the 5 most relevant document chunks, then sends the question along with those chunks and the conversation history to the LLM server on kaisens.fr. The LLM generates a response, which the backend stores in the Redis cache and saves in the session, then returns to the frontend which displays it to the visitor.
