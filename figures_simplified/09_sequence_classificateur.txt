sequence diagram - the Netethic platform sends a text to the Harassment Classifier API. The API first preprocesses the text by removing URLs, normalizing spaces, and converting to lowercase. It then runs 4 classification strategies in parallel. The first strategy sends the text directly to the LLM with a zero-shot prompt. The second strategy retrieves 3 relevant legal passages from ChromaDB and injects them into the prompt before calling the LLM. The third strategy uses fast keyword matching without calling the LLM at all. The fourth hybrid strategy tries the LLM first and falls back to keyword matching if the LLM fails. Once all 4 strategies return their results, the API selects the best one, checks if any label indicates suicidal ideation to set an urgency flag, logs everything, and returns the final labels, confidence score, strategy used, urgency flag, and latency to the platform.
