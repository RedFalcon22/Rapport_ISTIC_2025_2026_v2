state diagram - the Noa chatbot starts in an Idle state. When a message arrives it moves to Intent Classification, which routes to one of four paths: if the intent is chitchat it sends a predefined response and goes back to Idle; if the intent is forbidden it sends a refusal and goes back to Idle; if the intent is a reservation it enters a multi-step FSM that collects the user's email, then phone number, confirms the reservation, and returns to Idle; if the intent is factual it checks the Redis cache, and if there is a hit it returns immediately to Idle, otherwise it searches ChromaDB for relevant chunks, generates a response with the LLM, updates the cache and the session, then returns to Idle.
