state diagram - the Support Assistant starts idle, then waits for the user to select a category. Once a category is selected it waits for a question. From there three flows are possible: the main pipeline, an appointment flow, or an incident report flow. In the main pipeline the message goes through a guardrail check, spell correction, query rewriting, document search, and emotional detection. If distress is detected the system enters an emotional support flow that identifies the signal type, sends an empathetic response or an urgent alert if suicidal ideation is present, and guides the user to Netethic support resources. If no distress is detected a normal response is generated. The appointment flow is a 3-step FSM that collects email, then phone number, then confirms the appointment. The incident flow collects a description, then a screenshot, then confirms the report. All flows return to the waiting-for-question state when done.
